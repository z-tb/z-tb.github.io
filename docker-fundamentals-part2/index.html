Issue #1: Python 3.6 Compatibility

Problem Identified

SyntaxError: future feature annotations is not defined
File "/usr/local/lib/python3.6/dist-packages/bs4/builder/__init__.py", line 1 from __future__ import annotations ^ SyntaxError: future feature annotations is not defined

Root Cause

  • Using Python 3.6 (deprecated)
  • Base image ubuntu:bionic (also deprecated)
  • Modern packages require newer Python features
> Key Learning: Deprecated base images lead to compatibility issues with current packages

Solution #1: Modern Base Image

Strategy

  • Switch from Ubuntu to Debian slim distribution
  • Use bookworm-slim (recommended for similar issues)
  • Multi-stage Docker build approach

Dockerfile Changes

# OLD - Deprecated base image # FROM ubuntu:bionic as app # NEW - Modern slim distro FROM debian:bookworm-slim AS app

Additional Optimizations

  • Remove apt-get upgrade (security best practice)
  • Remove python-minimal (included in slim)
  • Change libpython3-dev ‚Üí python3-dev
  • Use uppercase FROM and AS keywords

Issue #2: Installation Script Failure

Problem

ERROR: failed to build: process "/bin/sh -c cd seroba && /seroba/install_dependencies.sh" did not complete successfully: exit code: 1

Likely Source

pip3 install ariba pysam==0.15.0 pymummer==0.10.3 biopython pyyaml==5.4.1

Version conflicts due to outdated package versions from bionic era

Debugging Strategy

Container Debugging Approach:
  • Remove problematic line from Dockerfile temporarily
  • Build runnable container for debugging
  • Add vim to package list for editing
  • Use shell aliases for efficient container management

Development Workflow: Docker Aliases

Helpful Command Aliases

# Build commands
alias db='docker build -t seroba:tmb --progress=plain .'
alias dbn='docker build -t seroba:tmb --progress=plain --no-cache .'

# Container interaction
alias de='docker exec -it seroba:tmb /bin/bash' # shell on running container
alias dr='docker run -it seroba:tmb /bin/bash' # start container with shell

Issue #3: Externally Managed Environment

The Problem

error during build: externally-managed-environment

This environment is externally managed
To install Python packages system-wide, try apt install python3-xyz

Python Packaging

  • Debian prevents system-wide pip installs to avoid breaking OS python packages
  • venv could work, but might require updating the entire image layout

Container-Specific Solution

Philosophy:
üò¨ --break-system-packages not great for production

Applied Fix:
